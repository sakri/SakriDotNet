Search Engine Optimization 2018 round 1

While looking around at other front end developer portfolios for inspiration, I ran into a good post by Rafael Caferati (link below). My interest in SEO runs nowhere as deep as his, Anyone in this business should be aware of the basics. In the 2000s I ran a "flash blog" on this domain which was generating around 500 visits a day at its peak.

My Blogging days went down the drain with the demise of Flash, and my google ranking plummeted along with it. Googling "sakri" would lead a user to my website around page 6. To rectify this grave injustice I:

  • bought an ssl certificate, now all my pages use https
  • made sure all my social media profiles link to sakri.net
  • used vanillaJS with minimal external libraries etc. minified and entirely embedded in the app.html files to reduce server calls and shorten loads as much as possible. The apps are under 100kb each, (with a few images).
  • copied the metadata structure from some highly visible sites and adjusted to my own
  • registered a local business through google. It's a bit weird to have my home address visible to anyone, but I'm renting at the moment anyway.
  • This is the big one : Even though my "cards menu apps" contain all their text content in the apps html file, my build script also outputs html files for every card. For example : portfolio/sakri_dot_net.html; This file contains all the metadata, a generated nav with links to all the portfolio pages and apps and some other template content.
  • made sure I had sitemaps containing all these html files, with robots.txt pointing to them
  • ...and a few other minor efforts not worth the mention.

All this took some effort, but it has started to pay off :

  • google.com/search?q=sakri : first page!
  • google.com/search?q=sakri : first page!
  • google.com/search?q=sakri : first page!

The current goal is to reclaim my throne for "sakri" from the wikipedia article (Sakri : a census town in the Dhule District of India), and actually increase visibility to the point where I will no longer have to search for projects (yes, a man can dream ;) )

Fluffy Morphing Cloud on Codepen

I've had this idea kicking around for a while, and wanted to generate some fluffy clouds for a game I'm planning to make. The idea is to crete two random "cloud" shapes, then morph between them to simulate a "cartoonish" fluffy cloud with the illusion of "real" cloud behaviour.

The cloud shapes are generated by plotting points along an egg shape, using a random radius. This results in random enough shapes, without having to worry about intersecting lines. To somewhat flatten my clouds, I used an egg shape instead of a plain circle.

Here's a pen explaining the shape generation:

Random cloud shape generator on Codepen

The "fluff" are just circles placed along a grid over the cloud shape. Cells which are inside the cloud shapes path are stored. The final animation creates two such grids, and interpolates between the two. When the morph is compplete, a new "target" cloud is generated.

Sakri.net Stats Module!

The "core concept" of this website is : "show the world examples of what I'm good at, and like to do (and maybe land some similar projects)". I enjoy making custom data visualizations, so I needed an excuse to include some. The spur of the moment solution was an app showing useless information about a visitors "visit stats".

This evolved into a tongue in cheek effort to persuade increased interaction with the site. The data viz widgets are all built from scratch with Html5 Canvas, are very lightweight and performant. A pie chart shows 'cards visit' data, a line chart shows 'clicks history' and a list of progress bars shows 'badges' or 'achievements' : upon reaching 100% the user is treated to a 'pixel confetti' animation.

For such a small project, I chose to go with Vue.js
